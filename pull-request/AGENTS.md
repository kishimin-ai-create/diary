# Pull Request Draft Codex Instructions

These rules apply inside `pull-request/`.

## Writing Rules

- Use `.github/pull_request_template.md` as the structure reference.
- Explain why the change is needed before listing implementation details.
- Include relevant test or validation results.
- Use repository-relative paths only.
- Do not include local absolute filesystem paths.

## Scope

- Keep each PR draft focused on one logical change.
- Mention intentionally excluded scope when it helps reviewers.
- Reference related requirements, ADRs, issues, or review files when available.
