## Session 1 — Fix CI small test failure in Drizzle config test

CI was failing on `bun test small.test` with one failure in `drizzle.config.small.test.ts`, even though the same small test command passed locally.

### Root cause

The test imported `backend/drizzle.config.ts` to access `createDrizzleConfig`. That file also evaluates its runtime default export immediately:

```ts
export default createDrizzleConfig();
```

Locally this worked because `backend/.env` exists. In CI, `.env` is not present during small tests, so importing the module required `DATABASE_URL` before the test could pass its fake environment.

### Fix

The reusable config builder was moved into `backend/drizzle.config.factory.ts`.

`backend/drizzle.config.ts` now stays strict for Drizzle CLI usage, while `backend/drizzle.config.small.test.ts` imports the side-effect-free factory and supplies fake env values directly.

### Verification

- `.env`-free factory import smoke check — passed

- `bun run lint` — passed

- `bun run db:migrate` — passed
